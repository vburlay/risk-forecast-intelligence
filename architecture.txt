1. CORE PRINCIPLE
   ========================
   Architecture layers:

2. data_access → data layer (DuckDB)

3. domain logic → forecast / anomaly / risk / simulation

4. services → orchestration (business assembly)

5. ui → presentation layer (Dash)

6. artifacts → models

data → forecast → anomaly → risk → simulation → ui
